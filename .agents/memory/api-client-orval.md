# Generated API client (@workspace/api-client-react)

Generated by orval from `openapi.yaml` into `lib/api-client-react/src/generated/api.ts`
(+ `api.schemas.ts`). Consumed by both `artifacts/web` and `artifacts/movil`.

## Naming gotcha (cost ~2 detours)
- **Query hooks** (GET) are emitted as `export function useListX<...>(...)` /
  `export function useGetX<...>(id, ...)` — NOT `export const`. Grepping
  `export const use` MISSES them. Use `rg "export function use"` or
  `rg "use(List|Get)X"`.
- Every GET also emits a **plain async fn** (`export const listX = async ...`,
  `getX`) plus `getXQueryKey()` / `getXQueryOptions()`. Use the plain fn for
  imperative one-off fetches (e.g. mobile chat initial load); use the hook for
  declarative screens.
- **Mutations** are `export const useXxx = ...`.

## Mutation `mutate` variable shapes (verify per-endpoint in api.ts)
- Body-only: `{ data }` — e.g. createChatGroup, checkInAccreditation.
- Path + body: `{ id, data }` — e.g. sendGroupMessage, submitSurveyResponse.
- Path-only: `{ id }` — e.g. markNotificationRead.
- No vars (`void`): call `mutate()` with NO args — e.g. markAllNotificationsRead.
  Calling `mutate({})` is a TS2345 error.

**How to apply:** before wiring any generated hook, confirm the exact
`MutationFunction<..., VARS>` type and whether it's `export function` (query) or
`export const` (mutation) by grepping api.ts.
